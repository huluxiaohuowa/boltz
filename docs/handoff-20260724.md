## Persistent directories

The web stack must use a host-mounted project directory:

```bash
BOLTZ_DATA_HOST_DIR=/data/ssd/jhu/boltz-web/data
```

The compose file mounts it to `/data` and sets:

```bash
BOLTZ_CACHE=/data/model-cache
```

This means user projects, uploaded files, generated assets, task outputs, and
Boltz model/cache files are expected to survive container rebuilds and restarts.

## Model source

Do not rely on implicit downloads into a container-local home directory. Download
the mirrored Boltz-2 model files from ModelScope into the persistent cache:

```bash
pip install modelscope
modelscope download \
  --model huluxiaohuowa/boltz-2-mirror \
  --local_dir /data/ssd/jhu/boltz-web/data/model-cache
```

Expected files:

- `boltz2_conf.ckpt`
- `boltz2_aff.ckpt`
- `mols.tar`

No local model files matching `boltz2_*.ckpt` or `mols.tar` were found under the
local checkout or the shallow `/Users/hulu` search performed during handoff, so
no local model file was removed.
